# 论文深度Review：Variable-Size Self-Contained Blocks for Discrete Diffusion Language Models

## 1. 核心方法与贡献

本论文针对离散扩散语言模型(dLLMs)中块状半自回归解码的**训练-推理不匹配问题**，提出了"自包含性"(self-containedness)作为块边界确定的原则性准则。核心创新是提出Variable-Size Self-Contained Blocks (VSB)框架，通过比较有无未来上下文条件下的预测分布的KL散度来动态确定块边界，避免了固定大小或启发式边界(如标点符号)造成的过早token提交问题。该方法在理论上建立了自包含性与预测一致性的联系，在实验中验证了相比于固定块大小的解码方式的有效性。

## 2. 技术细节

### 2.1 核心问题建模

论文首先形式化了训练-推理不匹配：
$$p_\theta(x_i|\tilde{x}^t_{1:L}, t) \neq p_\theta(x_i|\tilde{x}^t_{1:q}, t)$$

其中左侧为训练时的全序列条件分布，右侧为块式解码时的截断条件分布。

### 2.2 自包含性度量

定义两种条件分布：
- **No-Future (NF)**：$P^{NF}_{\theta,i}(p,b) = p_\theta(x_i|x_{1:b})$ - 仅使用候选边界b之前的信息
- **Future-Aware (FA)**：$P^{FA}_{\theta,i}(p,b,q) = p_\theta(x_i|x_{1:q})$ - 使用完整块预算内的所有信息

自包含性得分定义为：
$$S(p,b) = \frac{1}{b-p}\sum_{i=p+1}^{b} D(P^{NF}_{\theta,i} \| P^{FA}_{\theta,i})$$

其中D为KL散度。得分越低，块越"自包含"。

### 2.3 训练与推理框架

**训练阶段**：引入No-Future截断对齐
$$L_{NF} = -\mathbb{E}_{(x,t,x^t)} \left[\frac{1}{t}\sum_{i=1}^{b} \mathbb{1}[x^t_i=M] \log p_\theta(x_i|x^t_{1:b})\right]$$

这使得训练分布与推理时的截断条件更一致。

**解码阶段**：动态边界选择
$$b^* = \arg\min_{b \in (p,q]} S(p,b)$$

其中q为块预算的上界。

### 2.4 理论贡献

论文证明了NF-FA散度与块内预测一致性的联系，建立了自包含性原则的理论基础，使得动态边界选择不再是启发式而是有原理支撑的。

## 3. 优点与局限

### 优点

1. **原理性强**：将"何时提交块"这一实际问题形式化为自包含性原则，避免了固定大小或标点符号等启发式方法的任意性。相比于仅在训练时对齐的现有方法，该工作在解码阶段也实现了对齐。

2. **理论完整**：提供了从动机出发→形式化定义→解决方案的完整逻辑链条，建立了NF-FA散度与预测一致性的理论联系，使方法有坚实的理论基础。

3. **实验验证充分**：论文通过多个数据集和与多个基线(固定块大小、标点符号边界、最近的对齐方法)的对比，证明了VSB在生成质量和延迟上的改进，特别是在处理复杂语义单元时的优势。

### 局限

1. **计算开销明显**：方法需要在推理时为每个候选边界计算FA条件下的预测分布，相比固定块大小的方法引入了额外的计算成本。论文虽然在实验中报告了速度数据，但该额外开销(计算两种条件分布的forward pass)可能限制其在低延迟场景中的实际应用。

2. **块预算W的设定缺乏指导**：虽然论文在所有位置使用固定的块预算W，但关于如何选择或自适应调整W的讨论不足。W太小会限制可能的块大小范围，W太大会增加计算成本。这个超参数的敏感性分析和选择策略需要更深入的探讨。

3. **方法适用性有限**：当前方法假设可以访问完整的块预算内容来计算FA分布，但在真实的流式生成场景中，目标序列长度L不确定，导致块预算的定义和FA条件的实现存在问题。论文对这类场景的处理方案较少。

# DPRM论文深度Review

## 1. 核心方法与贡献

本论文提出DPRM（Doob h-transform Process Reward Model），一个可插拔的token排序模块，用于改进扩散语言模型的生成顺序。DPRM通过Doob h-变换将终端奖励信息注入到token排序策略中，从confidence驱动的排序逐步过渡到process-reward引导的排序，避免了纯confidence排序的近视性问题。该方法以理论保证（O(1/N)收敛性、样本复杂度优势）和广泛的实验验证（7个领域），证明了token排序是扩散模型的一个基本控制轴。

## 2. 技术细节

### 核心理论框架
**Doob h-变换与exact DPRM策略：**
定义正调和函数 $h_t(s) := \mathbb{E}_{q_0}[\exp(\beta R(X_T)) | S_t = s]$，其中$R(X_T)$为终端奖励。exact DPRM策略采用reward-tilted Gibbs reveal law：

$$\pi^*_t(a|s) := \frac{q_0(a|s)\exp(\beta R^*_t(a;s))}{\sum_{a' \in A(s)} q_0(a'|s)\exp(\beta R^*_t(a';s))}$$

其中 $R^*_t(a;s) := \frac{1}{\beta}\log \mathbb{E}[\exp(\beta R(X_T))|S_t=s, S_{t+1}=s_a]$

### 在线近似算法
由于exact策略计算成本高，论文采用三层近似：
1. **分阶段分桶统计**：用(phase $\phi$, confidence bucket $b$)定义决策单元，维护 $N_{\phi,b}$（计数）和 $S_{\phi,b}$（累积指数奖励）
2. **过程奖励估计**：$\hat{R}_{\phi,b} := \frac{1}{\beta}\log\frac{S_{\phi,b}}{\max(N_{\phi,b},1)}$
3. **门控转移**：通过gate函数 $\eta_{t,i}(\phi,b)$ 在warmup阶段逐步从confidence过渡到reward-aware排序

**算法1关键步骤**：从base proposal采样shortlist → 计算confidence和DPRM综合得分 → 排序并应用host action → 观测终端奖励并更新桶统计

### 收敛性保证
- Soft-BoN近似以 $O(1/N)$ 速率收敛到exact DPRM目标（KL意义下）
- 在线bucketized controller以empirical-Bernstein速率追踪exact DPRM分数
- 在tractable stagewise optimization假设下，DPRM相对随机和纯confidence排序具有样本复杂度优势

## 3. 优点与局限

### 优点
- **方法的通用性与即插即用特性**：DPRM不修改host模型、损失函数或数据管道，仅替换排序策略，具有极强的适配性。在7个异构设置（预训练、后训练、测试时scaling、多模态蛋白质、单细胞基因、分子药物、DNA设计）均可应用
- **理论与实践的紧密结合**：从Doob h-变换的经典概率论基础出发，通过分阶段分桶的在线近似实现可行算法，并给出收敛速率、样本复杂度的理论保证
- **实验结果显著且全面**：在语言任务上取得一致改进（GSM8K提升16.8%、MATH Hard提升8.1%），在科学领域暴露了有趣的性能边界trade-off（蛋白质RMSD降低17%、单细胞恢复率达75.92%）

### 局限
- **科学领域结果不一致性**：在蛋白质、分子和DNA生成任务中，排序改变的效果因具体评价指标而异。论文坦诚"不是所有质量指标都改进"，这说明reward设计和domain-specific适配仍需深入探索，某些应用场景中收益有限
- **理论与实践的gap**：论文基于tractable stagewise optimization假设证明样本复杂度优势，但该假设的实际满足情况、warmup阶段时长、桶粒度的选择等关键超参数的敏感性分析较缺乏。在线估计中的非平稳性、bucket coarsening偏差的实际影响量化不足
- **计算开销与可扩展性问题**：虽然相比exact策略有改进，但shortlist采样、桶统计维护、gate函数计算仍引入额外开销。论文未充分分析在更大规模模型、更长序列上的效率表现，以及与其他高效排序启发式的计算成本对比
